# i18n Design — English-First with Multilanguage Support

**Date:** 2026-03-18
**Status:** Approved

## Context

OpenTidy is going open-source. The entire app must be translated to English as the default language, with easy multilanguage support for future translations.

## Decisions

### 1. i18n Library: react-i18next

- **Frontend**: `i18next` + `react-i18next` — industry standard, JSON files, `useTranslation()` hook
- **Backend**: No i18n library — all messages hardcoded in English. Can be added later.
- Translation files live in `apps/web/src/i18n/locales/{en,fr}.json`

### 2. Status Constants Migration

| Old (French)   | New (English)  |
|----------------|----------------|
| `'EN COURS'`   | `'IN_PROGRESS'` |
| `'TERMINÉ'`    | `'COMPLETED'`   |
| `'EN ATTENTE'` | `'WAITING'` (display only) |
| `'faible'`     | `'low'`         |

- Defined in `packages/shared/src/types.ts`
- Backward-compatible parsing in `state.ts` (recognizes both old French and new English values)

### 3. Markdown Headers Migration

| Old (French)        | New (English)     |
|---------------------|-------------------|
| `## Objectif`       | `## Objective`    |
| `## Journal`        | `## Log`          |
| `## En attente`     | `## Waiting`      |
| `## Event declencheur` | `## Trigger`   |
| `## Mode Validation`| `## Confirm Mode` |
| `STATUT :`          | `STATUS :`        |
| `MODE : CONFIRM`    | `MODE : CONFIRM` (unchanged) |

- Parsing accepts both French and English headers
- New dossiers always use English headers

### 4. Claude Language Configuration

- New field `language?: string` in `OpenTidyConfig` (default: `'en'`)
- Injected into every system prompt: `"Always respond in {language}."`
- Claude writes state.md content (log entries, waiting reasons) in the user's language
- Section headers and status tags remain in English (machine-parsed)

### 5. System Prompts

All system prompts translated to English:
- Title generation (`title.ts`)
- Checkup analysis (`checkup.ts`)
- Memory injection/extraction/prompt (`agents.ts`)
- Dossier CLAUDE.md generation (`session.ts`)

Each prompt includes: `"Respond in {{language}}."` where language comes from config.

### 6. Suggestion/Gap File Format

New suggestions written in English format:
- `**Urgency:**` instead of `**Urgence :**`
- `## Why` instead of `## Pourquoi`
- `[RESOLVED]`/`[IGNORED]` instead of `[RÉSOLU]`/`[IGNORÉ]`

Parsing accepts both formats for backward compatibility.

### 7. Notification Messages

All Telegram and in-app notification messages in English.

## Files Changed

### Shared (`packages/shared/`)
- `types.ts` — status constants, UrgencyLevel

### Backend (`apps/backend/src/`)
- `workspace/state.ts` — parsing + writing with English headers
- `workspace/dossier.ts` — English state.md template
- `workspace/title.ts` — English system prompt
- `workspace/suggestions.ts` — dual-language field parsing
- `workspace/gaps.ts` — English markers with French fallback
- `launcher/session.ts` — English CLAUDE.md generation
- `launcher/checkup.ts` — English system prompt + messages
- `memory/agents.ts` — English prompts + language injection
- `notifications/telegram.ts` — English messages
- `config.ts` — language default

### Frontend (`apps/web/src/`)
- `i18n/i18n.ts` — i18next init
- `i18n/locales/en.json` — English translations
- `i18n/locales/fr.json` — French translations
- `main.tsx` — I18nextProvider
- All pages and components — `useTranslation()` hook

### Tests
- All test files using old French constants updated
