# Capabilities futures (hors scope V2)

Idées de capabilities à ajouter APRÈS que la fondation V2 est solide.
Chacune doit être un outil/skill modulaire que Claude appelle; pas de l'architecture.

## Idées

- **Appels téléphoniques**: Skill qui passe un appel, retranscrit, retourne le résultat
- **Réservations** (restaurant, RDV, etc.): Recherche + booking via web/téléphone
- **Conversations autonomes avancées**: Répondre aux messages en imitant le style de l'utilisateur
- **Shopping / achats en ligne**: Comparer, commander, suivre les livraisons
- **Gestion de voyage**: Vols, hôtels, itinéraires
- **Veille informationnelle**: Surveiller des sujets, résumer les nouveautés

## Principe

Tout futur capability = un skill que Claude appelle quand il en a besoin.
L'architecture V2 ne doit PAS être conçue spécifiquement pour ces features.
Elle doit juste être assez flexible pour qu'on puisse les brancher.
